## Known limitations before publish

- Only Brave Search is implemented as a live search provider in v0.1. Tavily, SerpAPI, and Bing have detection/status/parser coverage but remain unsupported adapters in this build.
- `web_search` live smoke test was not run because no provider API key was available in the environment.
- Browser automation requires a Chromium-family browser and a localhost CDP endpoint, either launched by the tool or manually started with remote debugging.
- Full live browser flow was not smoke-tested in this local readiness pass.
- CDP endpoints are intentionally restricted to localhost/loopback.
- `browser_eval` is disabled unless the server is started with `--allow-eval=true`.
- `fetch_url` intentionally blocks localhost/private/link-local/multicast/unspecified targets and only fetches HTTP(S) public URLs.
- Readability extraction is lightweight and may not match site-specific article extraction quality.
- Screenshot output is restricted to workspace-relative `.png` paths.
- The local build was generated without UPX compression; marketplace/GitHub release plans that require a small binary still need a UPX-enabled packaging run.
- `nullbot-web-mcp` is not currently a git repository in this workspace, so no git status/diff evidence is available locally.
